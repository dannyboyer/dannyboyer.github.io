BillETS

is a simple online platform for selling tickets for various shows. It is mostly written in the Java Enterprise Environment and is a school project. The main goal of the exercise was to design and implement a system that could handle a significant amount of requests in a short period of time.

Static prototype

The first step was to build a static, plain HTML website as a proof of concept. Technically, there is not much to say about this, except that we used Twitter's Bootsrap CSS framework and a minimalist approach for the design. Take a look at it here . Please, keep in mind that this is just a "static" version of the full website. If you are interested in trying the full featured version, you can check the source code on github.

JEE integration

After all the pages were ready, the JEE integration was done. The static pages were converted in ".JSP" format and the routing of the application was done using the Spring MVC framework, following the REST principles. Instead of leaving the show pages statically populated, an XML stub strategy was used. That way, it was easier to edit or add new shows to BillETS before the addition of a database.

Adding persistance

Currently, BillETS still use XML stubs for its collection of shows, but not for long. Soon, a persistence layer will be added to keep track of all the transactions, shopping carts and sessions. More details to come!
